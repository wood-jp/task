# Security Policy

## Supported Versions

Only the latest v1.x release receives security fixes. v1.x releases follow semantic
versioning — no breaking changes within the major version.

## Reporting a Vulnerability

Use GitHub's private vulnerability reporting:

https://github.com/wood-jp/task/security/advisories/new

Please include:
- task version
- Go version
- Description of the vulnerability
- Steps to reproduce
- Impact assessment

Do not open a public issue for security vulnerabilities.

## Response

We aim to acknowledge reports within a few business days. Fix timelines are
best-effort — this is a small open-source project.

## Disclosure

Once a fix is released, a GitHub Security Advisory will be published.
